# feature/cloud-sync 协作记录

- 目标：接入 Kelivo 私有同步服务，实现账户、设备、聊天与配置多端同步及云端附件。
- 当前阶段：客户端、生产 API 和云资源均已完成并通过线上闭环，等待快进合并到 `main`。
- 主要边界：LLM 对话请求仍由客户端直连供应商；同步服务只接收账户、配置、会话记录和附件元数据。
- 本地专属数据：代理、窗口、热键、平台运行状态、STDIO MCP、备份凭据及设备同步凭据不上传。
- 验收：代码生成、格式化、分析、现有测试、Android APK 构建和线上同步闭环均通过。
- 协作注意：其他进程修改聊天模型、Hive 字段、Provider 初始化或附件 marker 前，请先在此记录，避免产生并行迁移冲突。
- 已完成：ChatMessage 增加稳定 turnId 与生成状态，修复新消息 groupId 默认值；AssistantMemory 增加可确定性迁移的 syncId。
- 已完成：发送与重生成共享 turnId，完成、取消、错误和崩溃恢复会写入准确终态；旧消息按逻辑组一次性迁移。
- 客户端契约：固定 OpenAPI Generator 7.23.0，使用 `dart-dio + built_value` 保留判别联合；生成包位于 `dependencies/kelivo_sync_api_client`，禁止手改生成文件。
- 配置同步：已完成供应商、助手、记忆、世界书、快捷短语、搜索、网络 TTS、HTTP/SSE MCP 及显式用户偏好适配；代理、本地文件、STDIO MCP、备份和运行态保持本地专属。
- 同步协调：先拉取校验游标，再上传并追赶尾部；快照持久记录 seen keys，完整结束后权威删除已从云端物理清理的旧实体。
- 并发边界：未决 outbox 与已确认 changeSeq 保护本地二次编辑，不允许自身上传回声覆盖更新中的本地状态；快照重置采用服务端优先。
- 聊天适配：会话、轮次、消息的同步边界已建立；附件转换完成前不注册到运行时，避免带附件消息阻塞游标。
- 启动约束：`ChatService.init()` 已改为 single-flight；同步状态存储必须在它完成 Hive 初始化后再打开。
- 运行时状态：`CloudSyncProvider` 已接入应用根节点，持久恢复会话与设备级暂停状态，前台每 30 秒同步并在恢复前台时立即同步。
- 账户并发：登录、登出和暂停变更串行执行；登出先使会话失效并等待活动同步收口，销毁时在同步结束后再关闭 Hive box。
- 设置界面：移动端数据设置与桌面设置均已接入云同步；登录、状态、暂停、手动同步、设备撤销和退出共用同一内容组件，错误仅按本地化失败类型展示，不暴露令牌、内部 ID 或服务端原文。
- 附件同步：本地 marker 以账户、消息、类型、顺序、摘要和元数据生成确定性 UUID，完成 `prepare -> 无鉴权签名 URL PUT -> complete` 后才发布消息实体；远端按消息附件列表校验引用，下载到 `.part` 并通过大小与 SHA-256 校验后原子改名。
- 附件缓存：绑定状态按账户隔离持久化文件路径、mtime、大小、摘要、元数据和完成状态，避免周期同步重复散列与请求；消息更新不抢删服务端额外附件，孤儿回收留给后续服务端维护任务。
- 附件安全边界：签名 URL 使用独立且无 Bearer 的传输实例；附件与 blob 标识按 UUID 校验，下载目录在创建前相对固定云目录做越界检查；暂停同步会取消并等待活动传输收口后再重建暂停连接。
- 附件 P1 收口：仅用户消息末尾连续独立 marker 会进入附件转换，远端普通 user marker 以账户级正文指纹防止下一轮重解释；所有读写必须位于受管 upload 的已解析真实路径并拒绝路径链符号链接，缺失文件仅凭当前账户已完成绑定和服务端元数据核验后原位自愈；签名下载同时限制声明与实际字节为 100 MiB，生产地址强制 HTTPS 且签名传输拒绝重定向，最终落盘、孤儿清理与全量清理共用可重入临界区。
- Android 构建：统一使用 NDK 28.2.13676358；Windows 工程与 Pub 缓存跨盘会破坏 Kotlin 增量缓存，因此关闭 Kotlin/JVM 增量编译换取 release 构建稳定性；通用 APK 已完成自建证书验签。
- 生产接入：默认同步服务已硬切至 `https://kelivo.bemylover.top`，旧 Workers 地址全面弃用。
